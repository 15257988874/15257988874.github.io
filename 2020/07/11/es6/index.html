一、let/const

1.let

  • 不存在变量提升
  • 不允许重复声明
  • 暂时性死区(变量只在当前的块级作用域内有效)

2.const

对于简单类型的数据（数值、字符串、布尔值），值就保存在变量指向的那个内存地址，因此等同于常量。但对于复合类型的数据（主要是对象和数组），变量指向的内存地址，保存的只是一个指向实际数据的指针，const 只能保证这个指针是固定的（即总是指向另一个固定的地址），至于它指向的数据结构是不是可变的，就完全不能控制了

二、变量的解构赋值

注:结构赋值内部使用严格相等运算符（===），判断一个位置是否有值。所以，只有当一个数组成员严格等于 undefined，默认值才会生效

九、Module 的语法

与 AMD,CommonJS 语法不同的是,es6 模块注重静态化,使得编译时就能确定模块的依赖关系，以及输入和输出的变量。CommonJS 和 AMD 模块，都只能在运行时确定这些东西。比如，CommonJS 模块就是对象，输入时必须查找对象属性(前 2 者运行时加载,后者编译时加载)。由于 ES6 模块是编译时加载(静态加载)，使得静态分析成为可能。有了它，就能进一步拓宽 JavaScript 的语法，比如引入宏（macro）和类型检验（type system）这些只能靠静态分析实现的功能。

1.export

export 命令规定的是对外的接口，必须与模块内部的变量建立一一对应关系

  • 输出变量
  • 输出函数
  • 输出类

2.import

import静态执行,所以不能使用表达式和语句。 import 输入的变量都是只读的(对象除外,输入的变量不建议修改)

  • 基础
  • 默认
  • 混合

3.模块的整体加载

除了指定加载某个输出值，还可以使用整体加载，即用星号（*）指定一个对象，所有输出值都加载在这个对象上面

4.export default

export default 命令其实只是输出一个叫做 default 的变量，所以它后面不能跟变量声明语句

5.export 和 import 的复合写法

如果在一个模块之中，先输入后输出同一个模块，import语句可以与export语句写在一起。

6.模块的继承

export *，表示再输出指定模块的所有属性和方法。注意，export *命令会忽略指定模块的 default 方法

7.跨模块常量

  • 常量不多的情况
  • 常量较多的时候,建议分模块存储常量

8.import()

用于动态引入模块,返回一个 Promise 对象
