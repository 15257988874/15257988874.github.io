hexo + github 搭建个人博客--widnow

前言

近些年来很多用户都喜欢使用 GitHub Pages 来搭建 Hexo 静态博客网站，最吸引人的是完全免费使用，而且非常稳定。
本文依照自己搭建的心得，尽可能简化操作步骤

1.环境搭建 (git/node)

Hexo基于Node.js,搭建过程中还需使用npm(node自带)和git,故需先安装git以及node

  • Node.js: 下载
  • git:下载

两者下载一路下一步默认安装 。
安装完成后， Win+R 输入 cmd 并打开，依次输入 node -v、npm -v 和 git --version 并回车，出现版本号即可

2.连接 Github

注册github账号，在任一空白处右键 =>选择 git bash here,设置用户名和邮箱

2.1 创建SSH密匙(有密匙可跳过)

输入 ssh-keygen -t rsa -C "GitHub 邮箱"，然后一路回车。

2.2 添加密匙

进入 C:\Users\用户名\.ssh 目录（要勾选显示“隐藏的项目”），用记事本打开公钥 id_rsa.pub 文件并复制里面的内容。

登陆 GitHub ，进入 Settings 页面，选择左边栏的 SSH and GPG keys，点击 New SSH key。

title 随便取个名字，粘贴复制的 id_rsa.pub 内容到 Key 中，点击 Add SSH key 完成添加。

2.3 验证连接

打开 Git Bash，输入 ssh -T git@github.com 出现 “Are you sure……”，输入 yes 回车确认。显示 “Hi xxx! You’ve successfully……” 即连接成功。

3.创建仓库

GitHub 主页右上角加号 -> New repository

  • Repository name中输入 用户名.github.io(切记必须与当前登录的用户名一致,o(╥﹏╥)自己弄的时候没看仔细，在这坑了好久)
  • 勾选 ‘Initialize this repository with a README’
  • Description选填

填好后点击 Create repository,创建后默认自动启用https,地址为 https://用户名.github.io

4.初始化hexo项目

新建一个文件夹用来存放hexo的程序,例如hexo-blog，打开文件夹，git bash here

4.1 安装hexo以及本地预览服务器

npm i hexo-cli -g,安装时间有点久,耐心等待一下

npm i hexo-server -S,本地预览服务器

6.绑定域名(可选)

6.1 购买域名

在阿里云,腾讯云等域名服务商处购买域名(o(╥﹏╥)o本人再试狗爹上买的 https://sg.godaddy.com/zh?ci=,因为便宜…)

6.2 腾讯云解析域名 参考链接

添加域名，选择解析，添加两条记录类型为CNAME的解析，一条主机记录为@，一条主机记录为www，记录值都为你的格式为http://用户名.github.io的地址，在域名管理中使用自定义的域名服务商(我使用的是腾讯云,地址填写f1g1ns1.dnspod.net ，f1g1ns2.dnspod.net)

6.3 创建CNAME文件

本地博客文件的source文件夹下创建一个CNAME文件，记住不要有文件后缀名。编辑CNAME文件，里面写你在第一步申请的域名，例如xxxx.cn，记住不要有www。

6.4 预览上传

在你本地博客文件里运行hexo g，再运行hexo d

9.常见问题

1.Hexo设置显示文章摘要，首页不显示全文

Hexo 主页文章列表默认会显示文章全文，浏览时很不方便，可以在文章中插入 <!--more--> 进行分段。
该代码前面的内容会作为摘要显示，而后面的内容会替换为 “Read More” 隐藏起来。

2.设置网站图标

进入 themes/主题 文件夹，打开 _config.yml 配置文件，找到 favicon 修改，一般格式为：favicon: 图标地址。（不同主题可能略有差别）

3.修改并部署后没有效果

使用 hexo clean 清理后重新部署。
